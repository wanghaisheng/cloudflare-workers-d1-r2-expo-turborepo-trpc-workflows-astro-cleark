## 🎯 使用指南

### 1. 创建验收标准
```bash
# 复制模板
cp .codex/templates/acceptance-criteria-template.md 验收标准.md

# 选择合适的验收类型模板
# 根据项目需求选择功能、性能、质量、数据、集成验收模板

# 填写验收标准
# 按照模板填写所有必需字段
```

### 2. 验证验收标准
```bash
# 运行验证脚本
python .codex/scripts/acceptance_criteria_validator.py 验收标准.md

# 检查验证结果
cat acceptance_criteria_validation_report.md
```

### 3. 执行验收测试
```bash
# 运行验收测试
python .codex/scripts/acceptance_test_runner.py 验收标准.md

# 检查测试结果
cat acceptance_test_report.md
```

## 🎯 验收标准设计原则

### 1. 明确性原则
- 验收条件必须明确无歧义
- 验收方法必须具体可操作
- 通过标准必须量化可测量

### 2. 可验证性原则
- 验收标准必须可测试
- 验证结果必须可重复
- 验证过程必须可记录

### 3. 完整性原则
- 验收标准必须覆盖所有重要方面
- 验收方法必须覆盖所有关键路径
- 验收工具必须支持所有验证需求

### 4. 可操作性原则
- 验收标准必须可执行
- 验收工具必须可获取
- 验证流程必须可自动化
